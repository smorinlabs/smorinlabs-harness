# Export formats

Every codesign page exports BOTH formats from both control bars. Don't make
the user choose — MD is for humans, JSON is for machines.

## JSON — the source of truth

The embedded spec re-emitted with the user's live state: every choice's
`selected` reflects the toggles at export time, every `note.value` and
`feedback.value` carries the typed text, and `exported_at` is stamped. Schema
identical to `spec-format.md` — an export IS a valid spec, so
`validate_spec.py` lints it and a v2 can be rendered straight from it.

## Markdown — the human decision record

Reads like a lightweight ADR: what was decided, what was rejected, why.
Pasteable into a PR, Slack, or `docs/decisions/`.

```markdown
# Dashboard redesign — pick a direction

> _Choices made: 4 of 9 · Exported 2026-07-16_

Toggle the pieces you want, then export your picks.

---

## sec-01-direction · Overall direction

Pick one — these are mutually exclusive.

- **`ch-01-a` ✓ Focused — one hero number + one chart**
- `ch-01-b` ✗ Split — hero + a compact 3-tile strip

> **Notes (`note-01`):** Focused matches the "check one number" job.

---

## Overall feedback

> Can we see a hybrid of ch-01-a and ch-01-b before committing?

---

_Generated by html-codesign · exported 2026-07-16 · 2 sections · 9 choices · 4 selected._
```

### Rendering rules

- Title is the single `# H1`; the stat line (`Choices made: N of M ·
  Exported DATE`) is an italic blockquote directly under it.
- Section headers: `## {section.id} · {section.title}` — ID first so it is
  unmissable when quoted in chat.
- Choice lines: bullet → backticked ID → ✓ / ✗ → label. Selected lines are
  **bolded whole** (bullet through label).
- Notes with content render as blockquotes titled with their ID; empty notes
  are omitted.
- Footer: italic provenance line naming the skill, export date, and counts.

## Filenames

```
{title-slug}-{ISO-date}.{md|json}     dashboard-redesign-2026-07-16.json
```

Same-day re-export of the same title appends a counter:
`dashboard-redesign-2026-07-16-2.json`. The template handles slugging and
the counter.
